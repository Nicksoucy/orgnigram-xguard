Nightly Knowledge Base Pipeline (23h → 07h30)

  IMAP            KB Analyzer         Haiku            Supabase          Aggregator       Approval Email      Nick
   |                  |                  |                  |                  |                  |               |
   |    ============ 23:00 — KB ANALYZER STARTS ============

   IMAP -----------> Analyzer         |                  |              895 headers from 20 folders
   |                  |                  |                  |                  |                  |               |
   |             Analyzer <-----------   |             Supabase              GET kb_emails (existing IDs)
   |             Analyzer           |  -----------> Supabase              0 already done → 895 new
   |                  |                  |                  |                  |                  |               |
   IMAP <----------- Analyzer         |                  |              Fetch body (1500 max)
   IMAP -----------> Analyzer         |                  |              Email bodies (truncated 800ch)
   |                  |                  |                  |                  |                  |               |
   |    -------- CLASSIFICATION LOOP (5 workers x 20 batch) --------
   |                  |                  |                  |                  |                  |               |
   |             Analyzer -----------> Haiku             |              Classify email (JSON prompt)
   |             Analyzer -----------> Haiku             |              Classify email (parallel x5)

   |             Analyzer           |  -----------> Supabase              UPSERT kb_emails (per email)
   |                  |                  |                  |                  |                  |               |

   |    -------- 10s pause between batches --------
   |                  |                  |                  |                  |                  |               |
   |             Analyzer           |  -----------> Supabase              INSERT kb_run_log (completed)
   |                  |                  |                  |                  |                  |               |
   |    ============ 06:00 — TOPIC AGGREGATOR STARTS ============
   |                  |                  |                  |                  |                  |               |
   |                  |                  |             Supabase -----------> Aggregator         GET all faq_topics + counts
   |                  |             Haiku <-----------   |             Aggregator         Merge 150 raw → 40 canonical
   |                  |             Haiku ----------->   |             Aggregator         [{topic_id, label, response, merged}]
   |                  |                  |             Supabase <----------- Aggregator         UPSERT kb_topics + examples
   |                  |                  |             Supabase <----------- Aggregator         PATCH kb_emails.topic_id

Topic Approval Workflow (Human-in-the-Loop)

    Nick              Email              Admin Page          Supabase          kb_topics          kb_approvals
     |                  |                     |                  |                  |                  |
     |    ============ 07:30 — MORNING EMAIL ARRIVES ============
     |                  |                     |                  |                  |                  |
     Nick <----------- Email                  |                  |                  |              15 new pending topics
     Nick   reads email   |                     |                  |                  |              Reviews categories + frequencies
     Nick ---clicks link--->               Admin Page         |                  |              Opens kb_admin.html
     |                  |                     |                  |                  |                  |
     |                  |                Admin Page -----------> Supabase              |              GET kb_topics (all)
     |                  |                Admin Page <----------- Supabase              |              40 topics (15 pending)
     |                  |                Admin Page renders grouped by category                            Cards with approve/reject/correct
     |                  |                     |                  |                  |                  |
     |    ======== APPROVE: Nick clicks "Approuver" on "inscription-en-ligne" ========
     |                  |                     |                  |                  |                  |
     Nick ---clicks--->                  Admin Page         |                  |                  |
     |                  |                Admin Page -----------> Supabase              |              PATCH kb_topics (approved)
     |                  |                Admin Page ----------->   |                  | -----------> Approvals POST kb_approvals (audit log)
     |                  |                Admin Page: toast("Approuve!")                                    UI updates instantly
     |                  |                     |                  |                  |                  |
     |    ======== CORRECT: Nick edits response + clicks "Corriger" ========
     |                  |                     |                  |                  |                  |
     Nick types correction in textarea                          |                  |                  |
     Nick ---clicks--->                  Admin Page         |                  |                  |
     |                  |                Admin Page -----------> Supabase              |              PATCH kb_topics (corrected + new response)
     |                  |                Admin Page ----------->   |                  | -----------> Approvals POST kb_approvals (correct + old + new)
     |                  |                     |                  |                  |                  |
     |    ======== REJECT: Nick clicks "Rejeter" on spam topic ========
     |                  |                     |                  |                  |                  |
     Nick ---clicks--->                  Admin Page -----------> Supabase              |              PATCH kb_topics (rejected)
     |                  |                     |             Supabase           | -----------> Approvals POST kb_approvals (reject)
     |                  |                     |                  |                  |                  |
     |    ============ RESULT: Validated KB grows daily ============
     |                  |                     |                  |                  |                  |
     |    Approved topics = official knowledge base for future auto-response system
     |    Corrected responses = Nick's expertise captured permanently
     |    Rejected topics = filtered out, won't appear again
